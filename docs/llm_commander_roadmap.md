# LLM 지휘관 개선 로드맵

목표 2개 — ① **성능**(포획↑) ② **의도대로 배정**(통제성·일관성). 접근이 다르다.

> 원칙(Context7·통념): **프롬프트/스키마 엔지니어링으로 베이스라인부터** → 부족하면 구조 개선 → 그래도 부족하면 파인튜닝. 평가는 항상 SCORE 하네스로 A/B(시드↑로 노이즈↓).

---

## A. 즉효 (신뢰성·통제성, 낮은 노력)  ← **바로 적용**

**A1. 스키마 ↔ 프롬프트 정합** ★가장 시급
- 프롬프트는 "`{cluster_id, ally_ids}`만 출력"인데 스키마는 `deploy_net/net_legs/radius_adjust`를 강제(OpenAI strict = 모든 필드 required). 이 모순이 소형모델 폭주(16384토큰 파싱실패)의 원인.
- → 스키마를 **실제 쓰는 필드로 최소화**(셀 모델이 안 쓰는 net_legs/deploy_net/radius_adjust 제거). 최소 스키마 = 안정·저비용·통제성↑.

**A2. 추론-우선 순서** (Context7 권장)
- `rationale`(생각)를 `deployments` **앞**으로 → 모델이 "생각 후 결정". 현재는 rationale이 마지막이라 근거 없이 결정 후 사후정당화. 스키마 필드 순서만 변경.

**A3. SIDE-MATCH/효율 원칙 복원**
- 재작성 때 빠진 "가장 가깝고·선회 최소·비교차" 원칙 복원 + 제공 신호(`to_clusters` dist/turn, `bearing_from_center`) 활용 명시. 교차 배정 재발 방지. (코드 2-opt와 belt-and-suspenders.)

---

## B. 구조 개선 (신뢰성·의도, 중간 노력)

**B4. 코드-LLM 역할 분담** ★가장 효과적
- LLM = **전략**(어느 클러스터 engage/hold, 포메이션 적응), 코드 = **기계적 최적화**(효율 매칭·비교차). 이미 2-opt 도입.
- 확장: 코드가 **"추천 배정"(헝가리안 최적)**을 계산해 LLM 입력에 힌트로 → LLM은 수용/조정만. 효율 보장 + 의도 반영. 소형모델의 불안정한 공간추론을 코드가 backstop.

**B5. 상태 신호 강화**
- 추가 후보: cluster별 `time-to-breach`(우선순위), 코드 `recommended_assignment`, "이 배가 이 클러스터 맡으면 교차 발생" 플래그. 숫자로 주면 LLM이 암산 없이 정답 선택.

**B6. 의도 표현 채널 강화**
- 자연어 `command`를 **최우선 존중**하도록 프롬프트 명시("사용자 지시가 규칙과 충돌하면 지시 우선"). 예: "왼쪽 먼저" → 반영.

---

## C. 체계적 개선 (지속)

**C7. 평가 루프** (구축됨) — SCORE 하네스로 변경 A/B, 시드↑, **의도 테스트케이스** 추가("좌측 우선 지시 → 좌측 배 먼저?").

**C8. few-shot 골든 예시** — 의도를 담은 2~3개(side-match·포메이션). 편향 주의(정상케이스 혼합). 현재 off → 정합 후 재도입 가치.

**C9. 모델 선택** — 로컬 `qwen2.5:7b`가 14b 대비 품질 동등·6배 빠름(벤치 검증). OpenAI는 `gpt-4o ≫ gpt-4o-mini`(구조적 출력 안정성).

---

## D. 장기 — 파인튜닝

**핵심 판단**: "좋은 plan"을 코드가 계산 가능하면 파인튜닝 불필요(코드를 쓰면 됨). 파인튜닝의 가치 = **코드로 표현 못 하는 판단**(자연어 의도, 모호한 트레이드오프, 포메이션 뉘앙스).

- **노력/시간**: 재사용 조각(상태 직렬화·코드 교사·SCORE 하네스·`build_messages`) 이미 있음 → **첫 모델까지 ~1~3일**(데이터 파이프라인 0.5~1일 + 학습 10~30분 잡).
- **데이터 생성 (3소스 혼합)**:
  1. **코드 교사**(bulk·자동·무료): 다양한 state → 헝가리안+2-opt+포메이션 target. 형식·효율 학습.
  2. **강한 LLM 교사 + 결과 필터**: gpt-4o plan → 시뮬 SCORE 좋은 것만 채택(rejection sampling). 판단 학습.
  3. **사람/의도 라벨**: 의도-조건 state에 원하는 plan 직접. 통제성 학습.
  - 형식: JSONL `{messages:[system, user=STATE json, assistant=PLAN json]}` (= `build_messages` 그대로).
- **개수** (Context7: 시작 50, 품질>양): 이 작업은 좁음(고정 스키마·배3·클러스터≤3·포메이션3) → **~100~200(형식+효율) + 50~100(의도) = 합 ~200~300 실용**, ~500+ 수확체감. 로컬 LoRA(qwen 7b)도 유사.

---

## 권장 순서
**A1 → A2 → B4 → A3** (노력 대비 효과). 특히 **B4**가 "의도대로 + 효율" 둘 다 잡음. 파인튜닝(D)은 A·B로도 의도 추종이 일관되지 않을 때만.
