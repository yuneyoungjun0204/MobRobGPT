# LLM 지휘관 — 배정 판단 로직

### GPT가 "어느 USV를 어느 적 무리에" 배정하는가

전장상태 → **GPT** → 배정 계획(구조적 JSON)

MobRobGPT · 2026

---

## LLM 지휘관이 결정하는 것

- **입력**: 전장상태(모선·적 클러스터·아군 USV) + 파생 기하값
- **모델**: **GPT** (구조적 출력 — JSON 스키마 강제, temperature=0)
- **출력**: `CommanderPlan`
  - `rationale`(판단 근거) · `deployments[{cluster_id, ally_ids}]` · `hold_ships`
- **결정 범위**: 어느 배가 어느 클러스터를, **HOLD(대기)**, **포메이션 적응**
- **비결정**: 정확한 경로·그물 위치·충돌회피는 하위(정책/시뮬)가 처리

> GPT는 **전략적 판단**만 — 기하·물리는 넘기지 않음

---

## 판단 흐름

```
전장상태 ─▶ 파생신호 계산 ─▶ [GPT] ─▶ CommanderPlan ─▶ 시뮬 주입
            (거리·선회·방위·         (rationale 먼저
             그물덮임·중복 등)         → 배정 결정)
```

- GPT가 **좌표 암산을 안 하도록** 필요한 숫자를 미리 계산해 넣어줌
- GPT는 그 숫자로 **판단**만 → 소형 모델도 안정적

---

## 입력 로직 ① — 파생 신호 (암산 제거)

전장상태에서 **미리 계산해 GPT에 제공**:

| 신호 | 의미 | GPT가 하는 판단 |
|---|---|---|
| `to_clusters` [{dist, turn}] | 배→각 클러스터 이동거리·선회량 | 최근접·최소선회 매칭 |
| `bearing_from_center` | 아군의 모선 기준 방위 | 같은 쪽 배 고르기 |
| `reachable_radius` | 먼저 요격 가능한 반경 | 커버 가능 여부 |
| `nearest_cluster_gap_deg` | 인접 무리와의 방위차 | 옹기종기면 1척이 묶음 |

---

## 입력 로직 ② — 안전/중복 신호

| 신호 | 의미 | GPT가 하는 판단 |
|---|---|---|
| `net_covered` | 이미 그물로 차단됨 | 그 클러스터 **건너뛰기** |
| `route_hits_net` | 경로가 설치 그물 관통 | 격침 위험 → **HOLD** |
| `cluster_covered_by_teammate` | 팀원 그물벽에 덮임 | 중복 → **HOLD** |
| `assigned_cluster` | 직전 배정 | **연속성** 유지 |

> 위험·중복·연속성을 **플래그로** 주어 GPT가 규칙대로 판단

---

## 출력 로직 — 최소 스키마 + 추론 우선

```json
CommanderPlan {
  rationale:   "판단 근거를 먼저 서술"      ← ① 생각
  deployments: [{cluster_id, ally_ids}]     ← ② 배정
  hold_ships:  [ally_id, ...]               ← ③ 대기
}
```

- **최소 스키마**: 출력은 `{cluster_id, ally_ids}`만 → 폭주/모순 제거, 통제성↑
- **추론-우선**: `rationale`를 **맨 앞**에 → "생각한 뒤 결정"(사후정당화 방지)
- **구조적 출력 강제**: 스키마 위반 불가 → 항상 파싱 가능

---

## 전술 원칙 로직 ① (연속성·중복·최소)

1. **연속성** — 배정·상태 일관 유지, 잦은 타겟 전환 억제
2. **그물 lock-in** — 전개중 그물은 완료까지 중단/재배정 금지
3. **교대 HOLD** — 직전에 HOLD한 배를 연속 HOLD 금지(정체 방지)
4. **1클러스터=1배** — 중복 배정 금지(근접 무리는 1척이 묶어 커버)
5. **최소 병력, 돌파 불허** — 최소 척수, 단 어떤 클러스터도 안 뚫리게

---

## 전술 원칙 로직 ② (충돌·재전개·효율)

6. **충돌 회피 / 전원 HOLD 금지** — 충돌 위험 시 가까운 배만 두고 나머지 HOLD, 단 최소 1척은 활동
7. **그물 최대 3회 재전개** — 그물 소진 아니면 새 위협에 재투입
8. **효율 side-match** — 최근접·최소선회·**비교차** 매칭
9. **좌우 side-order** — 왼쪽 무리 `0→1→2` / 오른쪽 무리 `2→1→0` (모선 가로지르기·충돌 방지)

---

## 배정 효율 로직 — side-match + 좌우 order

```
              위(북)  ← 코너일수록 규칙 강하게
      NW●                    ●NE
   (좌상단)                 (우상단)
        \                   /
   왼쪽: 0 → 1 → 2     오른쪽: 2 → 1 → 0
          \      [모선]      /
           같은 쪽 배가 같은 쪽 위협 담당
```

- **각 배를 자기 쪽에** → 중앙(모선) 가로지르기 없음 → **모선 충돌·격침 방지**
- `to_clusters`(거리·선회)로 총합 최소, `bearing_from_center`로 같은 쪽 매칭

---

## HOLD 로직 — 언제 배를 멈추나

배정은 유지하되 **이번 주기 전진만 멈춤** (`hold_ships`):

- **중복** — 팀원이 이미 그 클러스터를 차단(`cluster_covered_by_teammate`)
- **그물 관통** — 경로가 설치 그물을 가로지름(`route_hits_net`) → 격침 회피
- **충돌 임박** — 두 배가 곧 만남 → 덜 급한 배 HOLD
- **시간차** — 파상에서 다음 파를 위해 대기

> HOLD = "재배정 없이 제자리 대기" → **연속성 유지**하며 위험만 회피

---

## 포메이션 플레이북 로직

명령에 `[ENEMY FORMATION: <name>]`을 실어 **포메이션별 로직** 분기:

- **집중** — 단일 밀집 → 1척(벽이 무리 span), 나머지 예비
- **양동** — 다방향 페인트 → 전 클러스터 side-match, 전량 추격
- **파상** — 시간차 연속 → ★**그물 스태거**
  - 첫 파에 전량 소진 금지, **파마다 새 그물 1개**로 순차 대응

> 같은 배정이라도 **포메이션에 따라 전개 방식**이 달라짐

---

## 안전장치 로직

- **검증/정제(sanitize)** — 중복 ally·무효 id 등 **거부 대신 수리** → 항상 유효 계획
- **폴백** — GPT 실패/타임아웃/스키마 위반 시 **위협비례 휴리스틱**으로 방어(멈추지 않음)
- **코드 보정** — 배정 후 **교차 자동 스왑(2-opt)** 으로 남은 비효율/교차 제거

> GPT가 어떤 상황에서도 **방어가 끊기지 않도록** 다중 안전망

---

## 요약 — LLM에 들어간 로직

- **입력**: 거리·선회·방위·그물덮임·중복을 **미리 계산**해 암산 제거
- **출력**: `{cluster_id, ally_ids}` 최소 스키마 + **rationale 먼저**(생각→결정)
- **판단 로직**: 연속성 · lock-in · 최소병력 · 효율 side-match · **좌우 order** · HOLD · 포메이션 스태거
- **안전**: 검증·수리 · 휴리스틱 폴백 · 코드 교차보정

### GPT는 "누구를·어디에·언제"만 판단, 나머지는 규칙과 하위 레이어가 보장
